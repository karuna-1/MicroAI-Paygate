# Code of Conduct

MicroAI Paygate is an open source project for people who want to learn, review, and improve a security-sensitive payment gateway. Contributors are expected to keep discussions technical, respectful, and focused on making the project better.

## Expected Behavior

- Be respectful when reviewing code, asking questions, or disagreeing.
- Assume good intent, but be precise when security, correctness, or user safety is involved.
- Keep feedback actionable. Explain the impact and suggest a path forward.
- Respect contributor time by opening focused issues and pull requests.
- Do not post secrets, private keys, API keys, wallet seed phrases, or sensitive logs.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing another person's private information without permission.
- Repeated off-topic comments that block maintainers or contributors from working.
- Public disclosure of security vulnerabilities before maintainers have had a reasonable chance to respond.
- Submitting intentionally malicious code, dependency changes, workflows, or documentation.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or commits that violate this code of conduct. Maintainers may also temporarily or permanently block participants who repeatedly disrupt the project.

If you need to report a conduct issue, contact a maintainer through a non-public channel when possible. If no private channel is available, open a short issue asking for maintainer contact without including sensitive personal details.

## Scope

This code of conduct applies to project spaces, including GitHub issues, pull requests, discussions, reviews, and any project-linked communication channel.
